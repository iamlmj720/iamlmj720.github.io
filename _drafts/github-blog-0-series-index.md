> 이 글은 GitHub 블로그 만들기 시리즈의 **목차(허브)** 글입니다. 각 편을 따라가면 빈 화면에서 검색까지 되는 블로그가 완성됩니다.

## 이 시리즈는

블로그를 직접 만들면서 "이건 어디서 바꾸지?" 하고 헤맨 것들을, 미래의 나와 같은 길을 걷는 분들을 위해 차근차근 정리합니다.
minimal-mistakes 테마 기준이며, 한 편을 따라 하면 결과물 하나가 손에 잡히도록 구성했어요.

## 목차

### 1부. 시작하기
- **[#1] GitHub 블로그란? 시작 전 큰 그림** — Pages·Jekyll·테마의 관계와 전체 흐름
- **[#2] 로컬 환경 세팅 & 첫 배포** — Ruby/Bundler, `jekyll serve`, 저장소 연결

### 2부. 기본 운영
- **[#3] 첫 글 쓰기** — `_posts` 규칙, front matter, 이미지 넣기
- **[#4] 카테고리·태그·사이드바 구성** — 분류 만들고 메뉴에 띄우기

### 3부. 커스터마이징
- **[#5] `_config.yml` 길들이기** — 프로필·스킨·폰트·About
- **[#6] 레이아웃·HTML 손대기** — `_includes`/`_layouts` 직접 수정

### 4부. 운영 & 성장
- **[#7] 검색에 노출시키기** — 구글 서치콘솔·네이버 서치어드바이저·사이트맵
- **[#8] 방문자 분석 & 댓글** — Analytics, Disqus/utterances
