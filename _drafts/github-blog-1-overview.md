## 이번 편에서 할 것
- 용어 정리하고 전체 흐름을 머릿속에 그리기 (아직 설치는 안 함)

## GitHub 블로그를 이루는 3가지
- **GitHub Pages** — 저장소 파일을 웹사이트로 띄워주는 무료 호스팅
- **Jekyll** — 마크다운 글을 블로그 형태(HTML)로 변환해주는 정적 사이트 생성기
- **테마(minimal-mistakes)** — 디자인과 기능을 미리 만들어 둔 틀

## 왜 GitHub 블로그인가 (장단점)
- 장점: 무료, 글=파일(기록이 쌓임), 완전 커스텀 가능
- 단점: 초기 진입장벽(터미널/구조 이해), 동적 기능은 제한

## 전체 작업 순서 미리보기
- 환경 세팅 → 글쓰기 → 카테고리 → 커스터마이징 → 검색 등록
- (각 단계가 이 시리즈의 한 편씩)
